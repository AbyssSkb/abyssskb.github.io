MobileNetV3

激活函数

MobileNetV3 使用了 h-swish\mathrm{h\text{-}swish}h-swish：

h-swish[x]=xReLU6(x+3)6\mathrm{h\text{-}swish}[x] = x\frac{\mathrm{ReLU6}(x+3)}{6} h-swish[x]=x6ReLU6(x+3)​

首先来看一下 swishswishswish：

swish[x]=xσ(x)swish[x] = x \sigma(x) swish[x]=xσ(x)

当 x→+∞x\to +\inftyx→+∞ 时，σ(x)→1\sigma(x) \to 1σ(x)→1，swishswishswish 退化成 xxx，当 x→−∞x\to -\inftyx→−∞ 时，σ(x)→0\sigma(x) \to 0σ(x)→0，swishswishswish 等于 000。不难看出，swishswishswish 是 ReLU\mathrm{ReLU}ReLU 的 soft version。

既然有了 ReLU\mathrm{ReLU}ReLU ，那为啥还要发明 swishswishswish 呢

为了避免神经网络在训练时出现梯度爆炸的问题，人们设计激活函数时会约束其导数的绝对值 ∣dydx∣≤1\vert\frac{dy}{dx}\vert \le 1∣dxdy​∣≤1，而 ReLU\mathrm{ReLU}ReLU 天生具有这一优势，但是其有一个弊端，ReLU\mathrm{ReLU}ReLU 的导数并不连续，这将降低拟合的速度以及效果，因此诞生了 swishswishswish，其导数连续，在 x=0x = 0x=0 附近制造了一个平缓区。

那为什么还要发明 h-swish\mathrm{h\text{-}swish}h-swish

swishswishswish 中的 σ\sigmaσ 计算量大，导致延迟较大，而 h-swish\mathrm{h\text{-}swish}h-swish 中的 ReLU6(x+3)6\frac{\mathrm{ReLU6}(x+3)}{6}6ReLU6(x+3)​ 其实就是对 σ\sigmaσ 的分段近似拟合，从而减少计算量，降低延迟。

首先 ReLU6\mathrm{ReLU6}ReLU6 是对 ReLU\mathrm{ReLU}ReLU 的最大值加以限制的版本

ReLU6[x]=min⁡(6,max⁡(0,x))\mathrm{ReLU6}[x] = \min(6, \max(0, x)) ReLU6[x]=min(6,max(0,x))

当 x>3x > 3x>3 时，ReLU6(x+3)6=1\frac{\mathrm{ReLU6}(x+3)}{6} = 16ReLU6(x+3)​=1，此时 h-swish\mathrm{h\text{-}swish}h-swish 退化成 xxx，当 x<−3x < -3x<−3 时，ReLU6(x+3)=0\mathrm{ReLU6}(x+3) = 0ReLU6(x+3)=0，此时 h-swish\mathrm{h\text{-}swish}h-swish 恒等于 000，这很类似 ReLU\mathrm{ReLU}ReLU

当 −3≤x≤3-3 \le x \le 3−3≤x≤3 时，h-swish\mathrm{h\text{-}swish}h-swish 等于 x(x+3)6\frac{x(x+3)}{6}6x(x+3)​
